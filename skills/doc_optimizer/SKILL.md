# 문서 최적화 스킬 (Document Optimizer)

## 개요

프로젝트의 마크다운 문서를 자동으로 분석하고 최적화합니다. 
**완성도 기준**으로 문서를 검토하여:
- 📝 **내용 누락** 보완
- 🗑️ **중복** 제거
- 💡 **예시** 추가

## 지원하는 문서

- `README.md` - 프로젝트 메인 설명서
- `SOUL.md` - 프로젝트의 철학/가이드
- `CLAUDE.md` - 개발 환경 설정
- 기타 마크다운 문서

## 동작 원리

### 1️⃣ 문서 분석 (Analysis)

각 문서를 분석하여 다음을 확인합니다:

| 항목 | 확인 사항 |
|------|---------|
| **내용 누락** | 설명이 불완전한 섹션, 빠진 사용 예시, 미설명 개념 |
| **중복** | 같은 내용의 반복, 중복된 섹션 설명 |
| **예시 부족** | 실제 사용 사례나 코드 예제 미포함 |
| **구조 개선** | 논리적 흐름 개선, 계층 정리 |
| **명확성** | 모호한 표현, 개선 가능한 설명 |

### 2️⃣ 개선 (Enhancement)

- ✏️ 불완전한 섹션 확장
- 🔄 중복된 내용 통합
- 📌 실제 예시/코드 추가
- 🎯 핵심 정보 강조
- 📚 관련 섹션 연결

### 3️⃣ 자동 적용 (Auto-Apply)

- 최적화된 문서로 원본 파일 **자동 수정**
- 백업 생성 (선택사항)
- 변경 사항 요약 제공

## 사용 예시

### 예시 1: README.md 최적화

**입력:**
```bash
doc_optimizer를 사용해서 README.md 문서를 완성도 있게 최적화해줘
```

**과정:**
1. README.md 분석
2. 누락된 내용 발견 (설치 트러블슈팅 누락 등)
3. 중복 제거 (반복되는 설명 통합)
4. 예시 추가 (실제 사용 코드)
5. 파일 자동 수정
6. 변경 요약 제공

**출력:**
```markdown
✅ README.md 최적화 완료!

[변경 사항]
- 설치 트러블슈팅 섹션 추가
- 반복된 API 설명 통합 (2곳 → 1곳)
- 3개 실제 사용 예시 추가
- 구조 개선: 목차 재정렬

수정된 파일: C:\AI리더양성과제\README.md
```

### 예시 2: SOUL.md 최적화

**입력:**
```bash
SOUL.md를 완성도 기준으로 최적화해줘
```

**과정:**
1. 섹션별 완성도 평가
2. 빠진 영업 팁 추가
3. 실제 경쟁사 사례 추가
4. 시나리오 예시 확장
5. 자동 파일 수정

**출력:**
```markdown
✅ SOUL.md 최적화 완료!

[추가된 내용]
- 경쟁사 가격 전략 분석 사례 2개 추가
- 영업 상황별 대응 팁 3개 추가
- 실제 시장 분석 예시 추가
- 고객 대응 스크립트 추가

[개선된 섹션]
- "💡 Tips & Best Practices" 확장
- "🎬 영업 활용 시나리오" 2개 추가 사례
```

## 최적화 체크리스트

문서 최적화 시 다음을 확인합니다:

### 📝 내용 누락
- [ ] 주요 개념이 충분히 설명되었는가?
- [ ] 사용 사례/예시가 충분한가?
- [ ] 트러블슈팅 정보가 있는가?
- [ ] FAQ 섹션이 필요한가?
- [ ] 관련 링크가 충분한가?

### 🗑️ 중복 제거
- [ ] 같은 내용이 여러 곳에 반복되는가?
- [ ] 섹션 간 중복이 있는가?
- [ ] 예시가 중복되는가?
- [ ] 설명이 정말 필요한가?

### 💡 예시 추가
- [ ] 실제 사용 코드가 있는가?
- [ ] 구체적 사례가 포함되었는가?
- [ ] 전후 비교가 필요한가?
- [ ] 스크린샷/다이어그램이 필요한가?

### 🎯 구조 개선
- [ ] 목차가 명확한가?
- [ ] 섹션 순서가 논리적인가?
- [ ] 헤딩 레벨이 일관되는가?
- [ ] 강조(bold, italic)가 적절한가?

## 최적화 기준 (Optimization Standards)

### Level 1: 최소 (Minimal)
- 명백한 오타 수정
- 누락된 단어 추가
- 기본 포맷 정리

### Level 2: 표준 (Standard) ⭐ 기본값
- 부분적 예시 추가
- 중복 제거
- 섹션 확장
- 명확성 개선

### Level 3: 상세 (Comprehensive)
- 풍부한 예시 추가
- 트러블슈팅 섹션 추가
- FAQ 추가
- 다이어그램/표 추가
- 실제 사용 사례 추가

## 백업 및 안전성

- 🔒 **자동 백업**: 수정 전 원본 파일 백업 생성
  ```
  README.md.backup_2026-06-12T16-20-00
  ```
- ✅ **검증**: 수정 후 마크다운 문법 검증
- 📋 **변경 로그**: 모든 변경사항 상세 기록

## 제외 항목

다음은 최적화 범위에서 제외됩니다:

- ❌ 코드 스타일 변경 (코드 리뷰 참고)
- ❌ SEO/메타데이터 최적화
- ❌ 언어 통번역 (번역기 참고)
- ❌ 레이아웃/다이어그램 재설계 (디자인 도구 참고)

## 명령어

```bash
# 기본 최적화 (표준 레벨)

# 모든 문서 최적화

# 상세 최적화
doc_optimizer를 상세(comprehensive) 레벨로 설정해서 최적화해줘
```

## 주의사항

- ⚠️ 자동 수정되므로 변경 전 반드시 **백업 확인**
- ⚠️ 핵심 의미는 **절대 변경하지 않음**
- ⚠️ 작성자의 **의도 존중**
- ⚠️ 과도한 변경 주의 (필요시만 수정)
