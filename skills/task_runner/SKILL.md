# 작업 오케스트레이터 스킬 (Task Runner)

## 개요

3개의 커스텀 스킬을 **자동으로 순차 실행**하여 완전한 작업 흐름을 처리합니다.

```
┌──────────────────────────────────────────────┐
│         Task Runner (통합 오케스트레이터)      │
└──────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────┐
│   Step 1: issue_write                        │
│   → GitHub 이슈 생성 (#번호 자동 추출)       │
└──────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────┐
│   Step 2: issue_runner                       │
│   → Step 1에서 생성된 이슈에 자동 댓글 추가  │
└──────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────┐
│   Step 3: doc_optimizer                      │
│   → 문서 자동 최적화                         │
└──────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────┐
│      최종 결과 보고 (통합 리포트)             │
└──────────────────────────────────────────────┘
```

## 포함된 스킬

### 1️⃣ **issue_write** (Step 1)
- **기능**: GitHub 저장소에 새로운 이슈 생성
- **입력**: 제목, 설명
- **출력**: 생성된 이슈 번호, URL
- **역할**: 작업 추적 시작

### 2️⃣ **issue_runner** (Step 2)
- **기능**: GitHub 이슈에 댓글 추가
- **입력**: issue_write에서 반환된 이슈 번호, 댓글 내용
- **출력**: 댓글 추가 완료
- **역할**: 이슈에 진행상황/추가정보 기록

### 3️⃣ **doc_optimizer** (Step 3)
- **기능**: 프로젝트 문서 자동 최적화
- **입력**: 없음 (자동 감지)
- **출력**: 최적화된 문서 목록
- **역할**: 문서 품질 향상

## 사용 방법

### 기본 사용

```bash
task_runner를 사용해서 전체 작업 흐름을 자동으로 실행해줘

입력:
  - 이슈 제목: "새로운 기능: 실시간 알림 기능"
  - 이슈 설명: "사용자에게 중요한 뉴스가 수집되면 실시간으로 알림을 보내는 기능을 추가합니다"
  - 댓글 내용: "현재 디자인 단계입니다. 다음주 월요일까지 프로토타입 완성 예정"
```

### 출력 형식

```

========================================

[Step 1] ✅ issue_write 완료
├─ Issue 번호: #125

[Step 2] ✅ issue_runner 완료

│  ├─ README.md (47줄 변경)
│  ├─ SOUL.md (63줄 변경)
│  └─ CLAUDE.md (22줄 변경)
└─ 추가된 컨텐츠: 12 섹션, 12 예시

[최종 통계]
├─ 생성된 이슈: #125

├─ 최적화된 문서: 3개
├─ 총 변경 라인: 132줄
└─ 총 처리 시간: ~15초

[다음 단계]
└─ GitHub에서 Issue #125 확인
   문서 변경사항 Git에 커밋
```

## 동작 흐름 상세

### Phase 1: 사전 검증
```
1. 필수 입력값 확인 (이슈 제목/설명/댓글)
2. GitHub Token 검증
3. 문서 파일 존재 여부 확인
```

### Phase 2: 순차 실행
```
Step 1: issue_write 실행
  ├─ 입력: 제목, 설명
  ├─ 실행: GitHub API 호출
  └─ 출력: Issue #번호 추출

Step 2: issue_runner 실행
  ├─ 입력: Step 1의 이슈 번호, 댓글 내용
  ├─ 실행: GitHub API 호출
  └─ 출력: 댓글 추가 완료

Step 3: doc_optimizer 실행
  ├─ 입력: README.md, SOUL.md, CLAUDE.md
  ├─ 실행: 문서 분석 및 최적화
  └─ 출력: 수정된 파일 목록
```

### Phase 3: 최종 보고
```
1. 각 Step의 결과 수집
2. 통합 리포트 생성
3. 다음 단계 제안
```

## 에러 처리

### 에러 발생 시 동작

| 발생 위치 | 동작 | 결과 |
|----------|------|------|
| Step 1 (issue_write) | ⛔ 중단 | Step 2, 3 미실행 |
| Step 2 (issue_runner) | ⛔ 중단 | Step 3 미실행 |
| Step 3 (doc_optimizer) | ✅ 계속 | 파일별 에러 리포트 |

### 발생 가능한 에러

```
❌ GitHub Token 없음
   → GITHUB_TOKEN 환경변수 설정 필요

❌ 이슈 생성 실패
   → 저장소 접근 권한 확인
   → GitHub API 상태 확인

❌ 댓글 추가 실패
   → 이슈 번호 유효성 확인
   → 저장소 접근 권한 확인

❌ 문서 최적화 실패
   → 문서 파일 존재 확인
   → 마크다운 문법 검증
```

### 필수 입력

```json

  "issue_title": "새로운 기능 또는 버그명",

  "issue_comment": "댓글 내용"
}
```

### 예시

```

```

- 문서 최적화 수치

## 주의사항

- ⚠️ **순차 실행**: 각 Step이 끝난 후 다음 Step 실행 (병렬 불가)
- ⚠️ **에러 처리**: 한 Step 실패 시 이후 Step은 미실행
- ⚠️ **권한**: GitHub Token 필수 (repo 스코프)
- ⚠️ **성능**: 전체 실행시간은 약 15~20초
