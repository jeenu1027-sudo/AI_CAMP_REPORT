# 하네스 검증 규칙 (Verification Rules)

`harness.py`가 자동으로 적용하는 데이터 품질 검증 규칙 목록입니다.

---

## 1. 필수 섹션 존재 검증

data.json 내 아래 6개 섹션이 모두 존재하고 비어 있지 않아야 합니다.

| 섹션 | 설명 |
|------|------|
| `lme_prices` | LME 금속 가격 (구리, 아연, 납) |
| `exchange_rates` | 주요 통화 환율 (USD, JPY, EUR, CNY) |
| `steel_news` | 철강 산업 뉴스 |
| `competitor_news` | 경쟁사 동향 뉴스 |
| `market_info` | 시장 정보 |
| `policy_news` | 정책/규제 뉴스 |

**위반 시**: `bug` 라벨로 GitHub 이슈 자동 등록

---

## 2. 데이터 신선도 검증

`updated_at` 타임스탬프가 **현재 시각 기준 24시간 이내**여야 합니다.

| 조건 | 조치 |
|------|------|
| 24시간 이내 | 정상 |
| 24시간 초과 | `bug`, `critical` 라벨로 GitHub 이슈 등록 |

**목적**: APScheduler가 멈췄거나 크롤러가 실패한 경우를 즉시 감지

---

## 3. LME 가격 이상값 검증

장기 시세 통계에 기반한 정상 범위를 벗어나면 경고합니다.

| 금속 | 정상 범위 (USD/톤) | 근거 |
|------|-------------------|------|
| 구리 (Copper) | $7,000 ~ $15,000 | 최근 5년 실제 시세 기반 |
| 아연 (Zinc) | $2,000 ~ $5,000 | 최근 5년 실제 시세 기반 |
| 납 (Lead) | $1,500 ~ $3,000 | 최근 5년 실제 시세 기반 |

**위반 시**: `bug` 라벨로 GitHub 이슈 등록, 수동 확인 요청

---

## 4. 환율 이상값 검증

원화 기준 정상 범위를 벗어나면 경고합니다.

| 통화 | 정상 범위 (KRW) | 비고 |
|------|----------------|------|
| USD | 1,100 ~ 1,700 | 달러/원 |
| JPY(100엔) | 800 ~ 1,100 | 100엔/원 |
| EUR | 1,400 ~ 2,000 | 유로/원 |
| CNY | 180 ~ 280 | 위안/원 |

**위반 시**: `bug` 라벨로 GitHub 이슈 등록

---

## 5. 검증 결과 판정 기준

| 결과 | 조건 | 하네스 종료 코드 |
|------|------|----------------|
| PASS | 이상 항목 0건 | `0` (정상) |
| FAIL | 이상 항목 1건 이상 | `1` (비정상) |
