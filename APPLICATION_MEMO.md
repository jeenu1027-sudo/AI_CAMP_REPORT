# 철강 선재 영업 업무 적용 메모 및 확장 계획

## 현재 적용 현황

### 실제 업무에서 활용하는 방법

**매일 아침 루틴 (8시 이후)**
1. Flask 대시보드 접속 → LME 가격 / 환율 / 뉴스 확인 (기존 30분 → 5분으로 단축)
2. 전일 대비 LME 구리 가격 변동 확인 → 고객 견적서 단가 판단 근거로 활용
3. USD/JPY 환율 확인 → 수입 원자재 가격 영향 분석

**주간 업무 활용**
- 경쟁사 뉴스 섹션 모니터링 → 시장 동향 파악
- 정책/규제 뉴스 → 수출입 관련 규제 변경 선제 대응

### AI 활용으로 개선된 점

| 항목 | 개선 전 | 개선 후 |
|------|---------|---------|
| 시장 정보 수집 시간 | 30~40분/일 | 5분 이내 (자동화) |
| LME 가격 추적 | 수동 웹 검색 | 대시보드 자동 갱신 |
| 경쟁사 동향 파악 | 불규칙적 | 매일 자동 수집 |
| 이슈 추적 | 이메일/메모 | GitHub Issues 활용 |

## 확장 계획

### 단기 계획 (1~3개월)

**1. 가격 알림 기능**
- LME 구리 가격이 전일 대비 ±2% 이상 변동 시 이메일 알림
- 환율 급변 시 즉시 알림 (USD/KRW ±20원 이상)
- 구현 방법: `smtplib` + APScheduler 조합

**2. 데이터 히스토리 저장**
- SQLite DB 도입으로 30일치 LME/환율 데이터 보관
- 차트 시각화 (Chart.js 활용)
- 고객 미팅 시 "지난 한 달 가격 추세" 바로 제시 가능

**3. 견적서 자동 계산 보조**
- 현재 LME 가격 + 환율 → 원재료 가격 자동 계산
- 마진율 설정 후 견적 단가 즉시 산출

### 중기 계획 (3~6개월)

**4. 고객별 맞춤 브리핑**
- 고객사별 관심 품목(선재, 봉강 등) 설정
- 개인화된 일일 브리핑 자동 생성 및 이메일 발송

**5. AI 시황 분석**
- 수집된 뉴스 + LME 데이터 기반 Claude API 시황 분석
- "이번 주 선재 시장 요약" 자동 생성

### 장기 계획 (6개월 이후)

**6. 모바일 앱 전환**
- React Native 앱으로 확장
- 출장 중에도 실시간 시황 확인

**7. 팀 공유 대시보드**
- 영업팀 전체가 접근 가능한 사내 서버 배포
- 팀원별 담당 고객사 가격 알림 개인화

## 배운 점 및 회고

### Claude Code 활용 경험

- **스킬 시스템**: 반복 작업을 스킬로 자동화할 수 있어 생산성 향상
- **이슈 기반 개발**: 문제 발견 → GitHub 이슈 등록 → 수정 → 닫기 사이클이 업무 추적에 효과적
- **에이전트 구조**: 수집(data_collector) + 검증(verifier) 역할 분리로 안정성 향상

### 철강 영업 도메인에서 AI가 특히 유용한 영역

1. **정보 수집 자동화**: 매일 반복되는 시황 정보 수집
2. **데이터 정규화**: 다양한 출처의 정보를 통일된 형식으로 변환
3. **이상 탐지**: 가격 급변 등 주의가 필요한 상황 자동 감지
